# 学習方針

## 目的

| # | 目的 |
|---|------|
| 1 | **Java Silver 合格**（Oracle Certified Java Programmer, Silver SE 11） |
| 2 | **JavaScript スキル向上**（実装を通じた実践的なスキル習得） |

---

## 学習要件

- Java Silver 合格に必要な知識を使いながら、**動作するアプリを作る**（必要に応じて複数）
- 画面作成に興味があるため、**動作する UI（画面）を含めて成果物完成**とする
- Java をバックエンド、HTML / CSS / JavaScript をフロントエンドとして組み合わせる

---

## 成果物の定義

- Java Silver の出題範囲をカバーした **Java アプリ**（1つ以上）
- **動作する画面**（HTML / CSS / JavaScript によるフロントエンド）
- フロントエンドとバックエンドが連携して動作すること

---

## AI が担当する範囲

| 担当項目 | 内容 |
|----------|------|
| **アプリの要件定義** | 何を作るか・機能一覧・学習項目との紐付け |
| **ディレクトリ構成の設計** | フォルダ・ファイル構成の提示 |
| **コードレビュー** | 実装後のコードに対してフィードバックを提供 |

---

## 学習ロードマップ（案）

### フェーズ 1: Java 基礎（〜 Silver 基礎範囲）

| トピック | 概要 |
|----------|------|
| 変数・データ型・演算子 | int / long / double / String / 型変換 / 演算 |
| 制御構文 | if / switch / for / while / do-while / break / continue |
| 配列 | 1次元・多次元配列 / Arrays クラス |
| コレクション | List / Set / Map / Iterator |
| 文字列操作 | String / StringBuilder / 主要メソッド |

### フェーズ 2: オブジェクト指向（Silver 核心範囲）

| トピック | 概要 |
|----------|------|
| クラス設計 | フィールド / メソッド / コンストラクタ / アクセス修飾子 |
| 継承・ポリモーフィズム | extends / super / オーバーライド / 動的束縛 |
| インタフェース | implements / default メソッド / 多重実装 |
| 抽象クラス | abstract / 抽象メソッド |
| 例外処理 | try-catch-finally / throws / checked / unchecked |
| ラムダ式・Stream API | 関数型インタフェース / filter / map / collect |
| モジュール | Java モジュールシステムの基礎 |

### フェーズ 3: アプリ開発（アウトプット）

| ステップ | 内容 |
|----------|------|
| アプリ要件定義 | AI が要件・機能一覧を提示 |
| ディレクトリ構成 | AI が構成を設計・提示 |
| Java 実装 | バックエンドロジックを自分で実装 |
| 画面実装 | HTML / CSS / JavaScript でフロントエンドを実装 |
| コードレビュー | AI がレビュー・改善提案 |
| 改修・仕上げ | レビュー結果を反映して成果物完成 |

## 技術スタック（想定）

| 区分 | 技術 |
|------|------|
| バックエンド | Java 11（SE） |
| フロントエンド | HTML / CSS / JavaScript |
| ビルドツール | javac（標準）/ Maven（必要に応じて） |
| サーバ | Java Servlet / HttpServer（軽量）/ Spring Boot（発展） |

---

## 備考

- アプリの候補は学習フェーズが進んだ段階で AI と相談しながら決定する
- Java Silver の出題範囲（公式シラバス）に沿ってトピックをカバーすることを優先する
- 画面は凝ったデザインより「動作すること」を最優先とする
