# アプリ02: TODO 管理アプリ

## 概要

タスク（TODO）を追加・編集・削除・完了管理できるWebアプリ。  
フェーズ2（オブジェクト指向）の知識を使って実装する。

---

## 対応する学習トピック（Java Silver 範囲）

| トピック | 使用箇所 |
|----------|----------|
| クラス設計 | `Todo` クラス / `TodoRepository` クラス |
| アクセス修飾子 | フィールドを `private`、getter/setter を `public` で設計 |
| インタフェース | `Repository<T>` インタフェースで CRUD を定義 |
| 継承・ポリモーフィズム | `TodoRepository` が `Repository` を実装 |
| 例外処理 | 存在しない ID への操作に独自例外をスロー |
| コレクション | `List<Todo>` / `Map<Integer, Todo>` でデータ管理 |
| ラムダ式・Stream API | フィルタリング・ソート・集計に使用 |

---

## 機能一覧

| # | 機能 | 説明 |
|---|------|------|
| 1 | タスク追加 | タイトル・期限・優先度を入力して追加 |
| 2 | タスク一覧表示 | 登録済みタスクを一覧表示する |
| 3 | タスク完了 | タスクを完了状態に切り替える |
| 4 | タスク削除 | タスクを一覧から削除する |
| 5 | タスク検索 | キーワードでタスクを絞り込む |
| 6 | フィルタ表示 | 「すべて / 未完了 / 完了済み」で表示切り替え |
| 7 | 優先度ソート | 優先度（高・中・低）順に並び替え |

|------|------|

## 非機能要件

- Java の `HttpServer`（`com.sun.net.httpserver`）を使った軽量 HTTP サーバとして動作する
- フロントエンドから `fetch API` でバックエンド（Java）の REST API を呼び出す
- データはメモリ上（`Map<Integer, Todo>`）で管理する（DB 不使用）

---

## API 設計

| メソッド | パス | 説明 |
|----------|------|------|
| `GET` | `/api/todos` | タスク一覧取得 |
| `POST` | `/api/todos` | タスク追加 |
| `PUT` | `/api/todos/{id}` | タスク更新（完了切り替え等） |
| `DELETE` | `/api/todos/{id}` | タスク削除 |

## クラス設計

### `Todo.java`

| フィールド | 型 | 説明 |
|------------|----|------|
| `id` | `int` | 自動採番 ID |
| `title` | `String` | タスクタイトル |
| `dueDate` | `String` | 期限（yyyy-MM-dd） |
| `priority` | `String` | 優先度（HIGH / MEDIUM / LOW） |
| `completed` | `boolean` | 完了フラグ |

### `Repository.java`（インタフェース）

```java
public interface Repository<T> {
    void add(T item);
    T findById(int id);
    List<T> findAll();
    void update(T item);
    void delete(int id);
}
```

### `TodoRepository.java`

- `Repository<Todo>` を実装
- `Map<Integer, Todo>` でデータを保持
- Stream API を使ったフィルタリング・ソートメソッドを追加

### `TodoNotFoundException.java`

- `RuntimeException` を継承した独自例外
- `findById` / `update` / `delete` で対象が存在しない場合にスロー

---

## 実装の流れ

1. `Todo.java` ・`Repository.java` ・`TodoRepository.java` の作成と単体テスト（コンソール）
2. `TodoNotFoundException.java` の作成と例外処理の組み込み
3. `TodoHandler.java` で HTTP リクエストのルーティングと JSON レスポンスの実装
4. `Main.java` で `HttpServer` を起動して動作確認
5. `web/` に HTML / CSS / JS を作成し、fetch API でバックエンドと連携
6. コードレビュー依頼
