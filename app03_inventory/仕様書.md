# アプリ03: 商品在庫管理アプリ

## 概要

商品の入庫・出庫・在庫照会・集計ができる在庫管理Webアプリ。  
フェーズ2の発展として、継承・インタフェース・ラムダ式・Stream API を中心に使って実装する。

---

## 対応する学習トピック（Java Silver 範囲）

| トピック | 使用箇所 |
|----------|----------|
| 継承 | `Product`（基底）→ `FoodProduct` / `GeneralProduct`（子クラス） |
| 抽象クラス | `Product` を `abstract class` として定義 |
| インタフェース | `Storable`（入出庫操作）/ `Reportable`（集計レポート出力） |
| ポリモーフィズム | `List<Product>` で異なる商品種別を一括管理 |
| 例外処理 | 在庫不足時に独自例外 `StockShortageException` をスロー |
| ラムダ式・Stream API | 在庫集計・カテゴリ別フィルタ・在庫切れ商品抽出 |
| コレクション | `List` / `Map`（カテゴリ集計）/ `Optional` |
| モジュール | パッケージ分割による疎結合設計 |

## 機能一覧

| # | 機能 | 説明 |
|---|------|------|
| 1 | 商品登録 | 商品名・カテゴリ・単価・初期在庫を登録 |
| 2 | 商品一覧表示 | 登録商品を一覧で表示（在庫数付き） |
| 3 | 入庫 | 指定商品の在庫数を増やす |
| 4 | 出庫 | 指定商品の在庫数を減らす（在庫不足は例外） |
| 5 | 在庫照会 | 商品名・カテゴリで検索 |
| 6 | 在庫切れ一覧 | 在庫数0の商品を抽出して表示 |
| 7 | カテゴリ別集計 | カテゴリごとの在庫総数・総額を表示 |

## 非機能要件

- Java の `HttpServer` を使った軽量 HTTP サーバとして動作する
- フロントエンドから `fetch API` でバックエンドの REST API を呼び出す
- データはメモリ上（`Map<Integer, Product>`）で管理する（DB 不使用）

## API 設計

| メソッド | パス | 説明 |
|----------|------|------|
| `GET` | `/api/products` | 商品一覧取得 |
| `POST` | `/api/products` | 商品登録 |
| `PUT` | `/api/products/{id}/stock-in` | 入庫（在庫数増加） |
| `PUT` | `/api/products/{id}/stock-out` | 出庫（在庫数減少） |
| `DELETE` | `/api/products/{id}` | 商品削除 |
| `GET` | `/api/products/out-of-stock` | 在庫切れ商品一覧 |
| `GET` | `/api/reports/category` | カテゴリ別集計 |

## クラス設計

### `Product.java`（抽象クラス）

| メンバ | 型 | 説明 |
|--------|----|------|
| `id` | `int` | 商品 ID |
| `name` | `String` | 商品名 |
| `category` | `String` | カテゴリ |
| `price` | `int` | 単価（円） |
| `stock` | `int` | 在庫数 |
| `getDescription()` | `abstract String` | 商品説明（サブクラスで実装） |

### `Storable.java`（インタフェース）

```java
public interface Storable {
    void stockIn(int quantity);
    void stockOut(int quantity) throws StockShortageException;
}
```

### `Reportable.java`（インタフェース）

```java
public interface Reportable {
    Map<String, Long> countByCategory();
    Map<String, Long> totalValueByCategory();
}
```

### `ProductRepository.java`

- `Map<Integer, Product>` でデータを保持
- 在庫切れ抽出：`stream().filter(p -> p.getStock() == 0)`
- カテゴリ集計：`stream().collect(Collectors.groupingBy(...))`

### `StockShortageException.java`

- `RuntimeException` を継承
- `stockOut` 時に在庫数が要求数を下回る場合にスロー

## 実装の流れ

1. `Product.java`（抽象）・`FoodProduct.java`・`GeneralProduct.java` の作成
2. `Storable.java`・`Reportable.java` インタフェースの作成
3. `StockShortageException.java` の作成
4. `ProductRepository.java` の実装（Stream API を活用）
5. `InventoryService.java` でビジネスロジックを実装
6. `Handler` クラスで HTTP リクエストを処理
7. `Main.java` で `HttpServer` を起動して動作確認
8. `web/` に HTML / CSS / JS を作成し、fetch API でバックエンドと連携
9. コードレビュー依頼
