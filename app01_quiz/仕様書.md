# アプリ01: Java 学習クイズアプリ

## 概要

Java Silver の基礎知識を問うクイズを出題し、回答の正誤を判定するWebアプリ。  
フェーズ1（Java 基礎）の知識を使って実装する。

---

## 対応する学習トピック（Java Silver 範囲）

| トピック | 使用箇所 |
|----------|----------|
| 変数・データ型 | 問題データの管理（int, String, boolean） |
| 配列 | 問題リストの格納 |
| コレクション | `List<String>` で選択肢を管理 |
| 制御構文 | 正誤判定の if 文、問題ループの for 文 |
| 文字列操作 | 回答比較（String.equals / equalsIgnoreCase） |

---

## 機能一覧

| # | 機能 | 説明 |
|---|------|------|
| 1 | 問題出題 | 登録された問題をランダムまたは順番に表示する |
| 2 | 回答選択 | 4択から回答を選択できる |
| 3 | 正誤判定 | 回答後に正解・不正解を即時表示する |
| 4 | 解説表示 | 正誤判定後に解説文を表示する |
| 5 | 結果表示 | 全問終了後に正解数・正答率を表示する |
| 6 | リトライ | 結果画面から再挑戦できる |

---

## 画面一覧

| 画面 | 説明 |
|------|------|
| スタート画面 | タイトル・問題数表示・スタートボタン |
| 問題画面 | 問題文・4択ボタン・進捗表示（例: 3/10） |
| 正誤表示画面 | 正解or不正解・正解の選択肢・解説 |
| 結果画面 | スコア・正答率・リトライボタン |

---

## 非機能要件

- サーバ不要。Java で問題データを JSON or 配列として持ち、静的 HTML + JavaScript で画面を構成する
- 問題データは Java クラスとして定義し、JSON 出力メソッドを持つ
- 画面は JavaScript で動的に描画する（シングルページ構成）
- JSON 文字列の組み立て時は、`"`、`\\`、改行（`\n`）を最低限エスケープして不正 JSON を防ぐ

## クラス設計

### `Question.java`

| メンバ | 型 | 説明 |
|--------|----|------|
| `text` | `String` | 問題文 |
| `choices` | `List<String>` | 選択肢（4つ） |
| `answerIndex` | `int` | 正解のインデックス（0〜3） |
| `explanation` | `String` | 解説文 |

| メソッド | 戻り値 | 説明 |
|----------|--------|------|
| `Question(String text, List<String> choices, int answerIndex, String explanation)` | - | 問題データを初期化するコンストラクタ |
| `getText()` | `String` | 問題文を返す |
| `getChoices()` | `List<String>` | 選択肢を返す |
| `getAnswerIndex()` | `int` | 正解インデックスを返す |
| `getExplanation()` | `String` | 解説文を返す |

### `Main.java`

- `Question` のリストを生成してコンソール or JSON ファイルに出力する
- 問題データは Java コード内にハードコーディング（初期段階）

### `QuizExporter.java`

- `exportQuestions(List<Question>)` で `List<Question>` を受け取り、JSON 形式の文字列を返す
- `String` / `StringBuilder` を使って手動で JSON を組み立てる（外部ライブラリ不使用）

## 工程と成果物

| 工程 | 実施内容 | 成果物 |
|------|----------|--------|
| 1. 要件確認 | 仕様書の目的・機能・画面要件を確定する | 本仕様書（要件確定版） |
| 2. 設計 | クラス設計・ディレクトリ構成・データ構造を決定する | クラス図（draw.io）、ディレクトリ構成案 |
| 3. 実装（Java） | `Question.java`、`Main.java`、`QuizExporter.java` を実装する | Java ソースコード、問題データ出力（questions.json） |

| 5. テスト | 問題出題、正誤判定、結果表示、リトライ動作を確認する | テスト結果メモ（確認項目と結果） |

- 各工程の完了時にレビューを実施し、次工程へ進む前に修正有無を確定する
- レビュー観点は「要件充足」「可読性」「Java Silver 学習観点」「動作確認」の4点とする
- レビュー結果は簡潔に記録し、改修後に再確認する

### クラス図の作成方針

- クラス図は draw.io で作成する
- 対象クラスは `Question`、`Main`、`QuizExporter` とする
- 関係（依存・利用）と主要フィールド／メソッドが分かる粒度で作成する

## 問題データ例（Java Silver 基礎）

```
Q: int型の変数に格納できる最大値はどれか？
  A. 2,147,483,647
  B. 9,223,372,036,854,775,807
  C. 32,767
  D. 2,147,483,648
正解: A（Integer.MAX_VALUE）
```
